基于 Go 的短视频 Feed 系统（后端 + 前端），包含账号、视频、点赞、评论、关注与 Feed 流；支持 Redis 缓存与 RabbitMQ 异步 Worker（API 进程与 Worker 进程可拆分部署）。

### 短视频信息流系统（仿抖音短视频）

*   **推拉混合**：普通作者写扩散至粉丝 Redis ZSET 收件箱，大 V 仅写发件箱读时合并拉取，规避百万粉丝级写放大；Redis 与 MySQL 游标（UnixMilli score）天然兼容，两侧可无缝降级切换
*   **热榜**：Redis ZSET 1 分钟滑动窗口 + 指数衰减因子（半衰期 24h），防止存量热门内容长期霸榜；P99 < 30ms
*   **多级缓存**：进程内 LRU + Redis 二级缓存，DB 查询量下降约 80%；分布式锁 + double-check 防缓存击穿
*   **Agentic 发布工作流**：视频上传后触发 LLM Orchestrator Agent，Agent 通过工具调用依次完成内容理解、热点对比、标题/标签生成、互动率预测与最优发布时机推荐，创作者 Human-in-the-loop确认后正式发布；整个流程复用现有 MQ Worker 架构，Agent 编排结果异步回写，不阻塞主链路
*   **单机压测（k6，1000 并发，60s）**：feed 接口 QPS 11,000+，P90 < 120ms，HTTP 错误率 0%；Redis 缓存热身后服务端响应 < 2ms（GIN 日志实测）
